Q & A

Welcome to the Q & A

  • Q1: Why did you decide to do this project?
    A1: We wanted to see what real-world finance is like and how we could apply computer science to it.

  • Q2: Why didn't you use real banks, and what was your workaround?
    A2: Real banks were out of scope, too expensive, and involve private data. So Tyler built a dummy bank instead.

  • Q3: Who worked on which parts of the project?
    A3: Tyler handled most of the back-end (and some front-end). Alexander (not here today) did most of the front-end (and some back-end). Hildur mapped transactions to categories and designed the poster.

  • Q4: What technology did you use, and what challenges did you face?
    A4: For the front-end, we used Expo (React Native with TypeScript). It was tricky—you really have to “do it the Expo way,” or things break.

  • Q5: How do you categorize and display user transactions?
    A5: First each transaction maps to a sub-category (e.g. “McDonald’s → fast food”), then that maps to a major category (e.g. “fast food → Food”).

  • Q6: How does the pie chart interaction work?
    A6: The pie chart shows only the major categories. Clicking a slice drills down to list all its individual transactions.

  • Q7: Which charting library did you use?
    A7: We used react-native-gifted-charts to render our charts.

  • Q8: What if you need to add or change categories later?
    A8: All mappings live in a simple config file. Updating that file automatically changes how transactions are grouped in the UI—no code rebuild needed.

  • Q9: How does the sign up work?
    A9: We make sure that the user has a real account by using google sign in or apple sign in if they have an ios device.

  • Q10: How does the 'forgot password' work?
    A10: Basically we had to set up an account to send emails. Once we had that done, we create an application sign in from the email credentials. That gives us access to use the email to send emails to any user who has forgotten their password and need to reset it.

  • Q11: Do you plan continue working on this project?
    A11: (Alexander) Yes, I think I'll keep working on this project and try actually using real data from my own personal bank statements to generate data and find out the best financial plan for myself. It'll be an opportunity to extract real-world data and use it to analyze and plan financial plans accordingly.

  • Q12: Why have a drawer navigation and a tab bar navigation?
    A12: The drawer is for things you probably won't go into as much like settings and adding goals. The tab is for easy accessibility to navigate to the most important pages. This includes the main page which gives a detail of the transactions, trends, and uploads for statements.
